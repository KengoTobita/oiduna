# ID Migration Guide

**Migration Date:** 2026-03-03
**Version:** Breaking Change - ID System Redesign

## Overview

The ID system has been completely redesigned to use server-generated 8-digit hexadecimal IDs instead of client-specified sequential IDs. This change supports back-to-back collaborative workflows where multiple clients share the same session.

## Breaking Changes

### Track Creation

**Before:**
```http
POST /tracks/{track_id}
Headers:
  X-Client-ID: alice_001
  X-Client-Token: <token>
Body:
{
  "track_name": "kick",
  "destination_id": "superdirt",
  "base_params": {"sound": "bd"}
}

Response:
{
  "track_id": "track_001",  // Client-specified
  "track_name": "kick",
  ...
}
```

**After:**
```http
POST /tracks
Headers:
  X-Client-ID: alice_001
  X-Client-Token: <token>
Body:
{
  "track_name": "kick",
  "destination_id": "superdirt",
  "base_params": {"sound": "bd"}
}

Response:
{
  "track_id": "a1b2c3d4",  // Server-generated (8-digit hex)
  "track_name": "kick",
  ...
}
```

### Pattern Creation

**Before:**
```http
POST /tracks/track_001/patterns/pattern_001
Headers:
  X-Client-ID: alice_001
  X-Client-Token: <token>
Body:
{
  "pattern_name": "main",
  "active": true,
  "events": [...]
}

Response:
{
  "pattern_id": "pattern_001",  // Client-specified
  "pattern_name": "main",
  ...
}
```

**After:**
```http
POST /tracks/a1b2c3d4/patterns
Headers:
  X-Client-ID: alice_001
  X-Client-Token: <token>
Body:
{
  "pattern_name": "main",
  "active": true,
  "events": [...]
}

Response:
{
  "pattern_id": "e5f6a7b8",  // Server-generated (8-digit hex)
  "pattern_name": "main",
  ...
}
```

### ID Format

| Item | Before | After |
|------|--------|-------|
| **track_id** | `track_001`, `track_002` | `a1b2c3d4`, `f9e8d7c6` |
| **pattern_id** | `pattern_001`, `pattern_002` | `e5f6a7b8`, `12ab34cd` |
| **session_id** | Not present | `56789abc` (auto-generated) |

- All IDs are now **exactly 8 characters**
- All IDs use **lowercase hexadecimal** (0-9, a-f)
- IDs are **generated by the server** with collision checking
- IDs are **unique within their scope** (session-wide)

## Migration Examples

### Python Client (Before)

```python
import requests

# Before - Client specifies IDs
track_id = "track_001"
pattern_id = "pattern_001"

response = requests.post(
    f"http://localhost:57122/tracks/{track_id}",
    json={
        "track_name": "kick",
        "destination_id": "superdirt"
    },
    headers={"X-Client-ID": "alice", "X-Client-Token": token}
)

response = requests.post(
    f"http://localhost:57122/tracks/{track_id}/patterns/{pattern_id}",
    json={
        "pattern_name": "main",
        "active": True,
        "events": []
    },
    headers={"X-Client-ID": "alice", "X-Client-Token": token}
)
```

### Python Client (After)

```python
import requests

# After - Server generates IDs, client extracts them
response = requests.post(
    "http://localhost:57122/tracks",
    json={
        "track_name": "kick",
        "destination_id": "superdirt"
    },
    headers={"X-Client-ID": "alice", "X-Client-Token": token}
)
track_id = response.json()["track_id"]  # Extract server-generated ID

response = requests.post(
    f"http://localhost:57122/tracks/{track_id}/patterns",
    json={
        "pattern_name": "main",
        "active": True,
        "events": []
    },
    headers={"X-Client-ID": "alice", "X-Client-Token": token}
)
pattern_id = response.json()["pattern_id"]  # Extract server-generated ID

# Use IDs in subsequent operations
requests.patch(
    f"http://localhost:57122/tracks/{track_id}",
    json={"base_params": {"gain": 0.8}},
    headers={"X-Client-ID": "alice", "X-Client-Token": token}
)
```

### JavaScript Client (Before)

```javascript
// Before - Client specifies IDs
const trackId = "track_001";
const patternId = "pattern_001";

await fetch(`http://localhost:57122/tracks/${trackId}`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "X-Client-ID": "alice",
    "X-Client-Token": token
  },
  body: JSON.stringify({
    track_name: "kick",
    destination_id: "superdirt"
  })

```

### JavaScript Client (After)

```javascript
// After - Server generates IDs, client extracts them
const trackResponse = await fetch("http://localhost:57122/tracks", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "X-Client-ID": "alice",
    "X-Client-Token": token
  },
  body: JSON.stringify({
    track_name: "kick",
    destination_id: "superdirt"
  })
});
const { track_id } = await trackResponse.json();  // Extract server-generated ID

const patternResponse = await fetch(
  `http://localhost:57122/tracks/${track_id}/patterns`,
  {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "X-Client-ID": "alice",
      "X-Client-Token": token
    },
    body: JSON.stringify({
      pattern_name: "main",
      active: true,
      events: []
    })
  }
);
const { pattern_id } = await patternResponse.json();  // Extract server-generated ID
```

| Operation | Before | After |
|-----------|--------|-------|
| Create Track | `POST /tracks/{track_id}` | `POST /tracks` |
| Get Track | `GET /tracks/{track_id}` | `GET /tracks/{track_id}` (unchanged) |
| Update Track | `PATCH /tracks/{track_id}` | `PATCH /tracks/{track_id}` (unchanged) |
| Delete Track | `DELETE /tracks/{track_id}` | `DELETE /tracks/{track_id}` (unchanged) |
| Create Pattern | `POST /tracks/{track_id}/patterns/{pattern_id}` | `POST /tracks/{track_id}/patterns` |
| Get Pattern | `GET /tracks/{track_id}/patterns/{pattern_id}` | Unchanged |
| Update Pattern | `PATCH /tracks/{track_id}/patterns/{pattern_id}` | Unchanged |
| Delete Pattern | `DELETE /tracks/{track_id}/patterns/{pattern_id}` | Unchanged |

- [ ] Remove all hardcoded track_id values from client code
- [ ] Remove all hardcoded pattern_id values from client code
- [ ] Update track creation to use `POST /tracks` (not `/tracks/{track_id}`)
- [ ] Update pattern creation to use `POST /tracks/{track_id}/patterns` (not `/patterns/{pattern_id}`)
- [ ] Extract `track_id` from track creation response JSON
- [ ] Extract `pattern_id` from pattern creation response JSON
- [ ] Store extracted IDs for use in subsequent API calls
- [ ] Update ID validation to check for 8-character hexadecimal format
- [ ] Test collaborative scenarios with multiple clients

## Benefits

1. **Collision-Free**: Server-generated IDs eliminate client-side coordination
2. **Collaborative**: Multiple clients can create tracks/patterns without conflicts
3. **RESTful**: Follows standard REST conventions (POST without ID in URL)
4. **Secure**: Prevents ID prediction and enumeration attacks
5. **Scalable**: 16^8 = 4.3 billion possible IDs per type

## Backward Compatibility

**There is no backward compatibility.** This is a breaking change that requires all clients to be updated.

Old client code will fail with:
- `405 Method Not Allowed` for `POST /tracks/{track_id}`
- `405 Method Not Allowed` for `POST /tracks/{track_id}/patterns/{pattern_id}`

## Session-Scoped IDs + Flat Pattern API (2026-03-03)

**Version:** Major redesign - Session-scoped IDs and Flat Pattern API

### Breaking Changes

#### 1. ID Scope: Global → Session

IDs are now unique within each session, not globally:

| ID Type | Before | After |
|---------|--------|-------|
| track_id | Global unique (8-digit) | **Session unique (4-digit)** |
| pattern_id | Global unique (8-digit) | **Session unique (4-digit)** |
| session_id | 8-digit | 8-digit (unchanged) |

**Impact**: Different sessions can reuse the same track_id "0a1f"

#### 2. ID Length: 8-digit → 4-digit

```python
# Before
track_id: "a1b2c3d4"  # 8-digit hex
pattern_id: "e5f6a7b8"  # 8-digit hex

# After
track_id: "0a1f"  # 4-digit hex
pattern_id: "3e2b"  # 4-digit hex
```

**Benefits:**
- More human-friendly (like PIN codes)
- Faster to type and communicate
- Session-scoped reduces collision probability
- Session deletion automatically releases IDs

#### 3. Pattern.track_id field added

```python
# Before
class Pattern:
    pattern_id: str
    pattern_name: str
    # No track_id

# After
class Pattern:
    pattern_id: str
    track_id: str  # Added - which track owns this pattern
    pattern_name: str
```

**Enables:**
- Flat Pattern API (no track_id in URL)
- Pattern movement between tracks (track入れ替え)

#### 4. Pattern.archived field added (Soft Delete)

```python
class Pattern:
    active: bool = True
    archived: bool = False  # Added - soft delete flag
```

**Two-axis control:**

| State | active | archived | Meaning | UI Display |
|-------|--------|---------|---------|------------|
| **Playing** | true | false | Currently playing | Visible |
| **Muted** | false | false | Temporarily silent | Visible (grayed) |
| **Archived** | false | true | Not in use | Hidden |
| **Restored** | true | false | Playing again | Visible |

**Operations:**
```bash
PATCH /patterns/{id}  {"active": false}   # Mute
DELETE /patterns/{id}                     # Soft delete (archived=true)
PATCH /patterns/{id}  {"archived": false}  # Restore
```

#### 5. Flat Pattern API added

**New flat endpoints** (no track_id in URL):

```bash
# List all patterns
GET /patterns
GET /patterns?include_deleted=true

# Get pattern by ID (no track_id needed)
GET /patterns/{pattern_id}

# Create pattern (track_id in request body)
POST /patterns
Body: {"track_id": "0a1f", "pattern_name": "main", ...}

# Update pattern (can move between tracks)
PATCH /patterns/{pattern_id}
Body: {"track_id": "3e2b", "active": false, "archived": false, ...}

# Soft delete pattern
DELETE /patterns/{pattern_id}
```

**Existing hierarchical API maintained:**
```bash
GET /tracks/{track_id}/patterns
GET /tracks/{track_id}/patterns/{pattern_id}
POST /tracks/{track_id}/patterns
PATCH /tracks/{track_id}/patterns/{pattern_id}
DELETE /tracks/{track_id}/patterns/{pattern_id}
```

### Migration Examples

#### Pattern Movement (楽器入れ替え)

**Before:** Cannot move patterns between tracks

**After:**
```bash
# Move kick pattern from track A to track B (change instrument)
PATCH /patterns/3e2b
Body: {"track_id": "0b2c"}

# The pattern now plays on a different track/instrument!
```

#### Soft Delete + Restore

**Before:** Hard delete only

**After:**
```bash
# Archive pattern (soft delete)
DELETE /patterns/3e2b
# Pattern still exists with archived=true

# Check archived pattern
GET /patterns/3e2b
# → {"pattern_id": "3e2b", "archived": true, ...}

# Restore pattern
PATCH /patterns/3e2b
Body: {"archived": false, "active": true}
```

#### List All Patterns (Flat API)

```bash
# Get all active patterns across all tracks
GET /patterns

# Get all patterns including archived
GET /patterns?include_archived=true
```

### Python Client Example (After)

```python
import requests

BASE_URL = "http://localhost:57122"
headers = {"X-Client-ID": "alice", "X-Client-Token": token}

# Create track (4-digit ID)
track_resp = requests.post(
    f"{BASE_URL}/tracks",
    json={"track_name": "kick", "destination_id": "superdirt"},
    headers=headers
)
track_id = track_resp.json()["track_id"]  # e.g., "0a1f" (4-digit)

# Create pattern (4-digit ID)
pattern_resp = requests.post(
    f"{BASE_URL}/tracks/{track_id}/patterns",
    json={"pattern_name": "main", "active": True, "events": []},
    headers=headers
)
pattern_id = pattern_resp.json()["pattern_id"]  # e.g., "3e2b" (4-digit)

# Mute pattern (flat API)
requests.patch(
    f"{BASE_URL}/patterns/{pattern_id}",
    json={"active": False},
    headers=headers
)

# Soft delete pattern
requests.delete(
    f"{BASE_URL}/patterns/{pattern_id}",
    headers=headers
)

# List all patterns (flat API)
all_patterns = requests.get(
    f"{BASE_URL}/patterns",
    headers=headers
).json()
```

### Use Cases

#### Use Case 1: Track Movement (楽器入れ替え)

```bash
# Create kick pattern on kick track
POST /tracks/0a1f/patterns
Body: {"pattern_name": "groove"}

# Decide you want this pattern on snare track instead
PATCH /patterns/3e2b
Body: {"track_id": "0b2c"}

# Pattern now plays on snare track!
```

#### Use Case 2: Pattern Library (Archive)

```bash
# Archive old patterns without deleting
DELETE /patterns/3e2b  # archived=true
DELETE /patterns/4f1c  # archived=true

# List only active patterns (UI clean)
GET /patterns
# Returns patterns with archived=false only

# Browse archived patterns later
GET /patterns?include_archived=true

# Restore a pattern you want to reuse
PATCH /patterns/3e2b
Body: {"archived": false, "active": true}
```

#### Use Case 3: Quick Mute/Unmute

```bash
# Mute pattern without deleting
PATCH /patterns/3e2b
Body: {"active": false}

# Unmute
PATCH /patterns/3e2b
Body: {"active": true}
```

### Session Lifecycle

```python
# Session created with _id_generator
session = Session()  # session._id_generator created

# IDs are session-scoped
track_id = session._id_generator.generate_track_id()  # "0a1f" (unique in this session)

# Different session can reuse same ID
session2 = Session()  # session2._id_generator created
track_id2 = session2._id_generator.generate_track_id()  # Could also be "0a1f"

# Session deleted → _id_generator deleted → IDs automatically released
del session  # All IDs released
```
