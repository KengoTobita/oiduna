## 🔍 各層の役割（要約）

### External Interface: クライアント層
- **目的**: 外部からOidunaを操作
- **特徴**: HTTP通信のみ、他パッケージに依存しない
- **Rust移植**: 低優先度（各言語で実装）

### Layer 1: API層
- **目的**: HTTPリクエストをビジネスロジックに橋渡し
- **特徴**: 薄いラッパー、ビジネスロジックを持たない
- **Rust移植**: 低優先度（Python FastAPIが成熟）

### Layer 2: アプリケーション層
- **目的**: ビジネスルールの実装、検証ロジック
- **特徴**: Manager分離、Silent Failure防止
- **Rust移植**: 中優先度

### Layer 3: コア層
- **目的**: リアルタイム実行エンジン、通信プロトコル
- **特徴**: 高精度タイミング、ドリフト補正
- **Rust移植**: 最高優先度（パフォーマンスクリティカル）

### Layer 4: ドメイン層
- **目的**: メッセージスケジューリング、最適化
- **特徴**: O(1)高速検索、送信先非依存
- **Rust移植**: 中優先度

### Layer 5: データ層（Foundation）
- **目的**: システム全体の基盤としてのデータ構造定義とバリデーション
- **特徴**: Pydantic BaseModel、型安全性、他の層に依存しない最下層
- **Rust移植**: 最優先（データ構造は言語間共有）
- **Foundation概念**: すべての層がこの層のモデルを使用

## 📖 読み方のガイド

### 初めての方
1. このREADMEで全体像を把握
2. [Layer 5: データ層](./layer-5-data.md)で基本的なデータ構造を理解
3. [データフロー例](./data-flow-examples.md)で実際の動きを確認
4. 興味のある層のドキュメントを読む

### 特定の機能を実装したい方
1. その機能がどの層に属するか判断
2. 該当する層のドキュメントを参照
3. 依存関係を確認
4. 実装例を参考にコードを書く

### Rust移植を検討している方
1. [Layer 5: データ層](./layer-5-data.md) - 最優先
2. [Layer 3: コア層](./layer-3-core.md) - 次に重要
3. [Layer 4: ドメイン層](./layer-4-domain.md) - パフォーマンス重視
4. [依存関係マップ](./dependency-map.md)で影響範囲を確認
